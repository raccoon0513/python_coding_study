# Q1. 등변사다리의 높이

## 문제
길이를 알고 있을 때, 등변사다리의 높이를 구하고 싶어한다. 구름이를 도와 주어진 조건에서 등변사다리의 높이의 제곱을 네 배한 값을 구하여라.

## 입력
첫재 줄에는 등변사다리의 빗변 $K$와 두 대변 $D_1$, $D_2$의 길이가 공백을 두고 주어진다

입력값 : 1 1 1

목표 출력값 : 4

```python
# 문제 1 등변사다리의 높이
# -*- coding: utf-8 -*-
# UTF-8 encoding when using korean
import math
user_input = input()
K, d1, d2 = user_input.split(" ")
K = int(K)
d1 = int(d1)
d2 = int(d2)
h=0
h = ( K**2 - (abs(d1-d2) / 2 )**2 ) *4
print(int(h))
```

## 풀이과정
등변사다리 높이 공식
$h= \sqrt{K ^2-{(\frac{|D_1-D_2|}{2})}^2}$

결과를 넣었으나 정답으로 인정되지 않았기에

print(h)에 int( )를 씌워 형변환했습니다


# Q2. 택시 요금
## 문제
구름이는 친구들과의 약속을 위해 택시를 타려고 한다. 이때, 처음 거리 $C$ 까지 거리  $1$ 당 요금 $A$ , 그 이후로는 거리 $1$ 당 요금 $B$이다. 구름이가 거리 D만큼 택시를 타게 될 때, 구름이가 지불해야 하는 요금을 알아보자.

## 입력
첫째 줄에 $A,B,C,D$가 공백을 두고 주어진다.

입력 : 5 20 40 91

출력 : 1220

입력 : 64 23 92 86

출력 : 5504

```python
# 문제 2 택시 요금
# -*- coding: utf-8 -*-
# UTF-8 encoding when using korean
user_input = input()
A,B,C,D = map(int, user_input.split())
pay = 0
if D < C :
	pay = D * A
else : 
	pay = (C * A) + ((D-C) * B)
print(int(pay))
```

# 풀이과정
keypoint : D가 C보다 짧을 수 있음
map() 함수를 통해 미리 입력을 정수형으로 형변환


# Q3. 바느질

## 문제
구름이는 바느질을 하고 있던 중, 실이 조금 부족하다는 것을 깨달았다. 미리 정해둔 바느질 구멍에 따라 순서대로 바느질을 이어가야 하지만, 실의 길이가 모자라기 때문에 바느질 구멍 하나를 건너뛰고자 한다. 처음과 마지막 구멍을 건너뛰면 바느질의 모양이 많이 틀어지므로, 처음과 마지막을 제외한 구멍 하나를 건너뛸 예정이다. 또한, 정확한 실의 길이를 재는 것은 어렵기 때문에, 두 바느질 구멍을 잇는 실의 길이는 바느질 구멍 사이의 좌표 차이의 절댓값을 합한 거리인 택시 거리를 사용하여 계산한다. 바느질 구멍 하나를 건너뛸 경우 구름이가 사용해야 하는 가장 짧은 실의 길이는 얼마인지 구하여라.

## 입력
첫째 줄에는 바느질 구멍의 개수 $N$이 주어진다.
다음 $N$개의 줄에는 바는질 구멍의 좌표 $X_i$, $Y_i$
가 공백을 두고 주어진다.
## 출력
첫째 줄에 바느질 구멍 하나를 건너뛸 경우 구름이가 사용해야 하는 가장 짧은 싫의 길이를 출력

입력값 :
```

0 0

```

목표 출력값 : 14

```python
# 문제 3 바느질
# -*- coding: utf-8 -*-
# UTF-8 encoding when using korean
# 문제 3 바느질
# -*- coding: utf-8 -*-
# UTF-8 encoding when using korean
import math
count = int(input())
arr = []
for i in range(count):
	arr.append(list(map(int, input().split())))

def manhattan_dist(p1, p2):
	x1, y1 = p1
	x2, y2 = p2
	return abs(x1 - x2) + abs(y1 - y2) 
	
def remove_test(arr):
	total_distance =0
	for i in range(len(arr)-1):
		total_distance += arr
	return total_distance

full_distance = 0
for i in range(len(arr)-1):
	full_distance += manhattan_dist(arr[i], arr[i+1])

results = []
for i in range(1, count-1):
	before = manhattan_dist(arr[i], arr[i-1]) + manhattan_dist(arr[i], arr[i+1])
	after = manhattan_dist(arr[i-1], arr[i+1])
	results.append(after - before)

print(full_distance + min(results))
```

## 풀이과정
택시 거리 : 맨하탄 거리를 뜻함

처음엔 루프 내에서 루프를 돌렸더니 runtime에러가 발생하여,

한번 루프를 통해 총 거리(total distance)를 구하고,

다른 루프를 통해 새 거리로 대체되었을때 줄어드는 거리(results)를

배열에 저장, min을 통해 가장 많이 빠지는 값을 추적,

full distance에 더하여 값을 도출함.

# Q5. 조별 과제 

## 문제
구름이와 바람이는 새싹 대학교에서 같은 강의를 듣고 있다. 그리고 오늘 둘이서 같은 조를 이루어서 조별 과제를 하게 되었다.

바람이는 구름이보다 과제를 하는 속도가 더 빠르다. 구체적으로, 각자 맡은 파트의 진행된 정도를 수치로 나타낼 때, 구름이는 하루 동안 $G$만큼 올릴 수 있으며,  올릴 수 있다. 이렇게 바람이가 구름이보다 과제를 하는 속도가 더 빠르기 때문에, 각자 맡은 파트의 진행된 정도를 최대한 비슷하게 올릴 수 있도록 다음과 같이 과제를 하기로 했다.

- 각자 맡은 파트에 대해 하루 동안 과제를 한다.
- 구름이의 파트보다 바람이의 파트의 진행된 정도가 더 높다면, 두 사람이 맡은 파트를 서로 바꾼다.
- 구름이의 파트와 바람이의 파트의 진행된 정도가 같다면, 바람이가 구름이의 파트의 진행된 정도를 $1$만큼 올려준다.

조별 과제 발표일까지 남은 일수가 주어질 때, 조별 과제 발표일 때 각자 맡은 파트의 진행된 정도를 알아보자. 이때, 조별 과제를 시작할 때의 두 사람이 맡은 파트의 진행된 정도는 둘 다 $0$이며, 하루도 빠짐없이 두 사람 다 과제를 한다고 가정한다.
## 입력
첫째 줄에 조별 과제 발표일까지 남은 일수 $N$, 구름이가 과제를 하는 속도 $G$, 바람이가 과제를 하는 속도 $B$가 공백을 두고 주어진다.

## 출력

입력값 : 3 6 9

목표 출력값 : 24 22 

입력값 : 5 3 4

목표 출력값 : 20 19

```python
# 문제 5 조별 과제
# -*- coding: utf-8 -*-
# UTF-8 encoding when using korean
N, G, B = map(int, input().split())

g_progression, b_progression = 0, 0
def exchange():
    global g_progression, b_progression
    g_progression, b_progression = b_progression, g_progression

for i in range(N):
    
    g_progression+=G
    b_progression+=B
    if g_progression < b_progression:
        exchange()
    elif g_progression == b_progression:
        g_progression +=1

print(g_progression, b_progression)
```

## 풀이과정

exchange 함수를 추가해 주기적으로 변경해주었다

변수 교환 로직에서 파이썬은
temp를 쓰는 대신 a, b= b,a를 사용 할 수 있다

지금생각해보니 함수를 만들 필요는 없던 것 같다

"일단 하루동안 숙제를 한다"
